# ステータスウィンドウ（物理）とは
この世界ではステータスウィンドウが存在し、誰でも自身のステータスを確認できる。しかし主人公だけはステータスウィンドウに特殊な性質があり、物理的な実体として世界に影響を与えることができる。

## ステータスウィンドウについて
- 「ステータスウィンドウ　オープン」と詠唱することで、誰でも自身のステータスを表示できる
- ステータスウィンドウの構成、表示内容などはすべて仕様として厳密に定められている
- ステーテスウィンドウは超常現象であり、人類は原理を解明できない上、仕様を変えることもできない。人類にとっては自然現象を超越した「この世の理」の一つとして、当たり前に存在している
- ステータスウィンドウはホログラム的に空中に浮かぶ。タッチパネルのようにタッチして操作できる
    - 本人以外は操作できない。本人の身体に反応するため、他者が無理やり操作させることもできる。指紋や指先に限定せず、どの部位で触れても操作できる
    - 表示は誰にでも見える。また半透明であるため裏面からも読める
- タイムアウトは 3 分であり、何も操作しなければ自動で閉じる。トップ画面の右上の✕ボタンで手動で閉じることもできる
- ステータスウィンドウは物理的な影響を及ぼさないし、光も発生させない。つまり光源としては使えない
- ステータスウィンドウは空気に描いているため、空気が乱れると表示も乱れる。表示が乱れすぎると自動で閉じられる。全体の n %が乱れると閉じられるが、これは先天的に決まっており **妨害耐性** という。n = 0.1 とシビアな人もいれば、n = 99.5 など雑に扱える人もいる
- ステータスウィンドウは同時に一つしか表示できない
- ステータスウィンドウの表示、操作、クローズに特別なコストはかからない。詠唱や操作するときの身体操作が必要なくらいである

## ステータスウィンドウ（物理）について

### 希少性
主人公ひとりだけが持っているユニークな性質である。

### 位置づけ
系統としてはスキルであり、つまりユニークスキルと言える。しかし世界のバグにより生まれたものであるため、人類はスキルとして捉えることができない。作中では、主人公はユニークスキルのつもりで捉えているが、他の人物は他の仕組みとして捉えるかもしれない。

スキル名は **物理化** であり、ステータスウィンドウに物理的な作用と使い心地を追加するものだが、バグにより人類が計測することはできない。しかし呼び名がないと不便であるため、主人公も偶然「物理化」という名前をつける。つまり主人公は、自身のこれを **ユニークスキル「物理化」** と捉えている。

また物理化されたステータスウィンドウを **物窓（モノマド）** と呼ぶ。主人公の命名であり、厨二病を少し患っているためカタカナにしている。

### 物理化の性質
ステータスウィンドウとの違い:

- モノマドはステータスウィンドウに常に適用される。モノマドでないステータスウィンドウを出すことはできない
- モノマドは好きな場所に、好きなサイズで、好きな数だけ生成できる
    - ただし生成可能な範囲、サイズ、量、精度などはスキルの練度と持ち主のレベルに比例する
- モノマドのデザインと表示内容は、通常のステータスウィンドウと同一である
- 複数存在するモノマドの表示内容はすべて同期され、どれかのモノマドを操作すれば、他のすべての表示もリアルタイムに同期される

実体:

- モノマドは実体がある
    - 厚さは 2cm 固定
    - 空間に固定されるため重さはない。また人類のいかなる手段でも動かすことはできない
    - いかなる物質よりも硬い。人類のいかなる手段を持ってしても傷すらつけることはできない。しかし反作用は超常的に調整されており、かけた力の分だけ返す
    - 液体、気体、原子、魔素を始めとするあらゆる物体を通さない。温度も通さない。たとえば防御のために主人公の周囲をモノマドで囲い切ると、温度も含めて鉄壁であるが、空気も通らないため窒息の原因になる
    - 光は通すが、有害になるレベルの強い光は自動で遮断される。つまり光を用いた攻撃に警戒する必要はない
- モノマド生成時、生成先の空間は上書きされる
    - いかなる物体であっても上書きできる
    - これに伴う空間の処理は超常的にゼロ秒で行われるため、圧力差や衝撃波、また上書きされた部分がどこに行くかといった余波は考えなくて良い。上書きされた部分はこの世から完全消滅する
- モノマドで主人公自身の身体を上書きすることはできない。また主人公の体内に生成することもできない
    - モノマド生成時にこれらの条件にひっかかった場合、その生成はキャンセルされる

モノマドの操作

- モノマドのクローズは詠唱で行うこともできる
- クローズは単一または複数のモノマドを指定して行うこともできる

表示のカスタマイズ:

- モノマドのデザインはカスタマイズできる
    - 何をどこに表示するか・しないかを自由に設定でき、設定も複数保存して切り替えが可能
    - ✕ボタンの表示位置や表示非表示も設定できる。たとえば誤って触れて閉じてしまうのを防ぐために、✕ボタンを非表示にしても良い
- モノマドのデザインはすべてのモノマドに適用され、変更した場合も即座にすべてのモノマドに反映される

モジュール化:

- モノマドのつくりかたをモジュール化しておくことができ、モジュール化するとモジュール名を詠唱するだけでつくれるようになる
    - 例1: 自分の身体を基準にして、自分から 1m 離して立法的に囲いきったパターンを「ガードキューブ」として登録。以後、ガードキューブと詠唱するだけでこのパターンでつくれる
    - 例2: 自分の左手指先の方向に、縦 100m、横 1m のモノマドを生成するパターンを「ビーム」として登録。以後、ビームと詠唱するだけでこのパターンでつくれる
- モジュール化の利便性のため、例1や例2で挙げたような基準、その他「親指を握り込んでいるときに」など豊富な条件が用意されており、自由に使える
- モジュール開発作業はプログラミング的であり、モノマド上で IDE を使ってプログラミングする。イメージは VSCode で良い。むしろ VSCode そのものであり、主人公が苦笑するほどである
    - これは意図的な設定であり、主人公の前世の水準が反映されていると（主人公自身が）気付ける。モノマドというバグが作為的なものであることを匂わせるための設定だ
